# Day 56 — Finale Gauntlet (Phase 3)

No new material, no fixed problem list — that's deliberate. The whole point of Phase 3 is
testing recognition *without* knowing the category in advance, which a pre-written list would
defeat.

**How to build today's mix (repeat this method each of Days 56-59):**
1. Open `content/error_log.md` — if anything there is still marked as a repeat offender
   (appears more than once, or its `Next Review Date` has passed more than once), pull 1-2 of
   those in preferentially. These are your actual weak points; the gauntlet should stress them.
2. Fill the rest of the slot (4-6 problems total) by picking blind across everything you've
   solved: open `content/spaced_review_deck.md` and `labs-go/archive/`, `labs-go/refresh_sprint/`,
   `labs-go/week_8/` … `week_16/`, and pick without reading the pattern name — e.g. close your
   eyes and land on a row, or have someone else (or a script) pick an index for you. The goal is
   genuine surprise: you should not know the category until you've read the problem statement.
3. Full pressure, no exceptions: hard timer per problem (use the same 20-25 min default), talk
   out loud, no hints, no notes, no peeking at old solutions.
4. Self-grade each attempt on: pattern-ID speed (did you recognize it fast, or flounder?),
   correctness, complexity, code cleanliness.
5. Log every result to `content/weekly_challenge_log.md` — same file the weekly self-challenges
   used, so the trend line spans the whole 60 days, not just this week.
6. Anything that fails today goes into `content/error_log.md` immediately, same as any other
   day — Phase 3 doesn't relax the logging discipline, it's the last chance to catch what's
   still weak before you consider the plan done.
